Up and running in 1 minute

  • 1️⃣ Open Reachy Mini Dashboard → Applications
  • 2️⃣ Enable "Show community apps"
  • 3️⃣ Install "Reachy Mini HA Voice"
  • 4️⃣ Home Assistant discovers automatically

How it works

Seamless integration

This Reachy Mini app uses ESPHome protocol to communicate with Home Assistant — no ESPHome device needed. Home Assistant discovers it via mDNS and adds the robot entities automatically. Voice commands are processed by your Home Assistant instance — STT, intent recognition, and TTS all happen there.

Capabilities

Everything you need for smart home control

Zero-configuration robot entities, built-in reactions, and auto-discovery via mDNS.

🎤

Voice Wake

Local wake word detection with MicroWakeWord. Say "Okay Nabu", "Hey Mycroft", or "Hey Jarvis" to activate.

🏠

Smart Home Control

Full Home Assistant integration. Control lights, switches, climate, media — anything in your Home Assistant.

🎯

Sound-Source Orientation

Direction-of-arrival data turns the robot toward the speaker on wake, with smooth head and body motion.

🎛️

Manual Pose Control

Home Assistant number entities control head pose, body yaw, and antennas, including smooth manual yaw hold.

😊

Expressive Motion

Built-in listening, thinking, speaking, timer, and emotion reactions with natural head sway and non-blocking motion during conversations.

🔊

Multi-room Audio

Sendspin protocol support. Sync audio playback with other speakers throughout your home.

⚡

Zero Configuration

Install and go. mDNS auto-discovery and built-in HA reactions mean the default experience works without extra setup.

🃏

Dashboard Card

Optional Lovelace card for Home Assistant. Real-time 3D visualization of robot pose and status.

🚀

Auto Release

Version-driven GitHub release workflow. Update pyproject/changelog, then release is created automatically.
